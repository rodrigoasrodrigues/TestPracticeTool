Estrutura do Pacote
Baixar modelo YAML 
# Estrutura recomendada do ZIP
# pacote.zip
#   ├── materia_questoes.yaml
#   └── images/
#       ├── q00001_enunciado.png
#       ├── q00001_explicacao.png
#       └── q00001_opcao_3.png

# Exemplo de YAML dentro do pacote
questions:
  - text: "Enunciado da questão"
    image_file: "q00001_enunciado.png"            # opcional
    explanation: "Explicação (só no gabarito)"    # opcional
    explanation_image_file: "q00001_explicacao.png"  # opcional
    correct: 3          # número da opção correta (1–5)
    options:
      - text: "Opção A"
      - text: "Opção B"
      - text: "Opção C"      # correta (correct: 3)
        image_file: "q00001_opcao_3.png"  # opcional
      - text: "Opção D"
      - text: "Opção E"

  - text: "Segunda questão..."
    correct: 1
    options:
      - "Opção A"      # também aceita o formato simples antigo
      - "Opção B"
      - "Opção C"
      - "Opção D"
      - "Opção E"
Regras
  • Cada questão precisa de exatamente 5 opções.
  • correct é o número (1–5) da opção correta.
  • explanation e explanation_image_file são opcionais e aparecem apenas no gabarito.
  • image_file também pode ser usado nas opções para associar imagens às respostas.
  • O YAML deve conter a chave raiz questions: e estar dentro do pacote .zip.
